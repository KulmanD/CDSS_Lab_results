# Evaluation Plan

The MVP must include synthetic data and expected outcomes so the rule engine can be checked objectively.

## Dataset Requirements

Create synthetic examples for:

- Normal anemia/glucose/kidney/lipid/CRP cases.
- Borderline cases.
- Abnormal cases.
- Missing optional markers.
- Historical trend cases.

Target at least 5-10 examples per implemented rule category before final submission.

## Expected Outcomes

Each evaluation row should include:

- Synthetic patient demographics.
- Current lab results.
- Optional historical lab results.
- Expected triggered rule IDs.
- Expected urgency level.
- Notes about why the rule should or should not trigger.

## Metrics

The evaluation script should report:

- Accuracy.
- Precision.
- Recall.
- F1-score.
- Per-rule confusion counts.

## MVP Acceptance Criteria

- Anemia, glucose, and kidney rules are covered by normal, borderline, abnormal, and missing-data scenarios.
- Lipid and CRP rules are covered by normal, borderline or moderate, abnormal, and high-urgency scenarios.
- Trend scenarios exist for anemia and kidney checks.
- No evaluation record contains real patient data.
- Expected outcomes are manually defined, not generated by an AI service.
